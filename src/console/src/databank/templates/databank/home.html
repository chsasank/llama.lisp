Execution Model

Johnaic ETL is designed around a direct execution model where the system behaves exactly as defined — no hidden transformations, no unnecessary abstraction layers. Once a source and target are configured, pipelines simply define how data should move between them.

During execution, the engine dynamically reads the schema from the source and verifies its existence in the target system. If the structure does not exist, it is created automatically. If it already exists, the system proceeds with data movement without interruption.

Data transfer is handled in controlled batches, ensuring stability even under failure conditions. Each batch is temporarily staged and only committed once integrity is guaranteed, eliminating the risk of partial writes or data loss.
